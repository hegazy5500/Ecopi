# EcoLoop Mainnet Acceptance Checklist

This file translates official Pi listing guidance into build and review tasks.

## Must satisfy before submission

- [ ] Developer account KYC completed
- [ ] App works inside Pi Browser
- [ ] UI is polished and fully functional
- [ ] User login uses Pi Authentication only
- [ ] Transactions are Pi-only
- [ ] No fiat pricing in UX
- [ ] No external redirect required for core flows
- [ ] Privacy policy limits data collection to functional minimum
- [ ] App branding does not misuse Pi trademarks
- [ ] App demonstrates real utility, not only concept screens
- [ ] Backend verifies Pi auth/payment lifecycle securely
- [ ] Error states and empty states are present on core screens

## Official criteria reflected here

Pi's published Mainnet listing expectations include:
- fully functional app with professional UI
- complete KYC verification
- avoid trademark violations
- use only Pi authentication
- Pi-only transactions
- avoid redirection to external sites
- limit data collection

- https://pi-apps.github.io/community-developer-guide/docs/gettingStarted/mainnetListingRequirements/
